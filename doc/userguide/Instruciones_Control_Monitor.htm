Instruciones de Control / Monitor

La interface Postilion GenericInterface responde a los siguientes comandos usando Postilion Monitor como se especifica a continuación:

TRACE

  • TRACE <Nombre de la Interchange> <Parámetro>

<Nombre de la Interchange>: El nombre asignado a la Interchange

<Parámetro>: ON/OFF/CLEAR/?

Si el parámetro es ON, el trace está encendido. Si el parámetro es OFF, el trace se deshabilita o apaga. Si el parámetro es CLEAR los traces que existan serán borrados. Si el parámetro es ? la interchange retorna si el trace está encendido o apagado.

Si el trace está encendido o habilitado, el archivo de trace muestra los mensajes de <Postilion>\Trace\AthNode\<Interchange>. Los archivos del rastro están en formato del HTML y se pueden ver con un browser del Internet tal como Microsoft Internet Explorer.

SIGN ON

  • SET <Nombre de la Interchange> <Parámetro>

<Nombre de la Interchange>: El nombre asignado a la Interchange

<Parámetro>: SIGN-ON

Este comando envia un mensaje administrativo 0800 de tipo SIGN ON hacia la entidad remota. Se identifica porque en el Trace el campo 70 contiene el valor de 001

KEY EXCHANGE

  • SET <Nombre de la Interchange> <Parámetro>

<Nombre de la Interchange>: El nombre asignado a la Interchange

<Parámetro>: KEYEXCHANGE

Este comando envia un mensaje administrativo 0800 de tipo NEW KEY hacia la entidad remota. Se identifica porque en el Trace el campo 70 contiene el valor de 162

ECHO TEST

  • SET <Nombre de la Interchange> <Parámetro>

<Nombre de la Interchange>: El nombre asignado a la Interchange

<Parámetro>: ECHO-TEST

Este comando envia un mensaje administrativo 0800 de tipo ECHO TEST hacia la entidad remota. Se identifica porque en el Trace el campo 70 contiene el valor de 301
